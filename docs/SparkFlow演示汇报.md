## 一、背景与问题

### 1.1 当前审图工作现状

配电网建设项目施工图审查是工程立项前的关键质控环节。以罗定供电局2025年中低压配电网第十批紧急项目为例：

| 维度 | 现状 |
|------|------|
| 审查项目数量 | 4个项目，涉及数十张图纸 |
| 主要审查内容 | 技术意见（设计规范、图纸一致性）+ 技经意见（造价合理性）|
| 当前作业方式 | 人工逐条核查评审意见表 → 对照图纸 → 手工填写回复 |
| 典型问题类型 | 建设规模数量校核、图纸标注缺失、材料清册内容不符等 |

### 1.2 痛点分析

```
❌ 人工逐图核查耗时长，评审专家精力被大量消耗在重复性核查上
❌ 评审意见与图纸之间的对应关系需要手工梳理，容易遗漏
❌ 审查过程不留痕迹，整改问题难以追踪
❌ 同类错误在不同项目中反复出现，缺乏规则沉淀机制
```

## 二、SparkFlow 系统介绍

### 2.1 产品定位

**SparkFlow** 是面向配电网/一次系统图审查场景的 **CAD 自动审图 CLI 工具**，核心链路为：

```
DWG 图纸 → CAD解析 → 结构化建模 → 规则检查 → 报告输出
```

**当前定位：** 本地 CLI 工具 + 批处理工作流，聚焦于"辅助人工审图、提升审图效率"而非完全替代人工。

### 2.2 核心能力架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        SparkFlow 功能架构                         │
├─────────────────┬────────────────────────────────────────────────┤
│  输入层          │  DWG/DXF 图纸 + 评审意见 XLSX + 规则集          │
├─────────────────┼────────────────────────────────────────────────┤
│  解析层          │  CAD 实体抽取 / DWG→DXF 转换 / 文字坐标提取      │
├─────────────────┼────────────────────────────────────────────────┤
│  建模层          │  设备识别 / 导线拓扑 / 连通图 / 电气图分析         │
├─────────────────┼────────────────────────────────────────────────┤
│  规则层          │  评审意见 → 规则自动提取 → 规则引擎执行            │
├─────────────────┼────────────────────────────────────────────────┤
│  输出层          │  审查报告 (JSON/MD/DOCX) + 整改问题清单           │
└─────────────────┴────────────────────────────────────────────────┘
```

### 2.3 审图能力分级

| 级别 | 能力 | 说明 |
|------|------|------|
| Level 1 | 图元抽取 | 实体类型统计、图层分析、关键坐标定位 |
| Level 2 | 设备对象识别 | 从图块+文本构建设备实例与接线端子 |
| Level 3 | 拓扑关系建模 | 导线拆段、连通分量分析、拓扑规则执行 |

## 六、系统能力边界说明

### 6.1 当前可自动化的审查类型

| 审查类型 | 自动化程度 | 说明 |
|----------|-----------|------|
| 图纸页面存在性核查 | ✅ **高度自动化** | 通过关键词匹配确认指定图纸页面存在 |
| 占位符/未定稿文字检测 | ✅ **高度自动化** | 检测 XXX、FXX 等未填写的占位符 |
| 设备文本关键词匹配 | ✅ **高度自动化** | 在图纸全文中定位目标文本 |
| 拓扑连通性规则 | ✅ **自动化** | 导线连通性、断路器配置等拓扑规则 |
| 数量核实（跨文档） | ⚠️ **需人工核查** | 说明书数量 vs 图纸数量，需跨文档对比 |
| 材料清册核查 | ⚠️ **需人工核查** | 依赖清册附件，超出图纸范围 |
| 造价调差系数 | ⚠️ **需人工核查** | 依赖造价预算文件 |

### 6.2 系统对人工审核的赋能方式

```
传统审核流程：
  评审专家 → 手工阅读评审意见 → 逐张查找图纸 → 记录核查结果 → 撰写整改清单

SparkFlow 辅助后：
  评审专家 → 运行流水线（5分钟内完成）→ 审阅自动通过条目（确认无误）
           → 重点核查 manual_review 条目 → 整改问题清单已自动生成
```

**效率提升点：**
- 评审规则从 XLSX 自动提取，无需手工录入
- 图纸文本全自动解析，提供精确证据定位
- 自动通过项无需人工复核，节省约 50% 核查时间
- 整改清单自动生成 `.md` / `.json` / `.docx` 三种格式
