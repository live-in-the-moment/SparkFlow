# 基于大模型和智能体的工程设计评审系统：融合方案

## 1. 系统定位

本系统不是“工程问答机器人”，而是面向电力设计院的工程设计自动评审、风险识别、整改闭环和知识资产沉淀平台。核心目标是辅助专业负责人完成设计说明、图纸、设备表、计算书、工程量清单等资料的合规性、完整性、一致性和经济性审查。

## 2. 融合后的总体架构

系统采用五层架构：

1. 用户交互层：仪表板、项目中心、智能评审台、报告中心、规范知识库、配置中心。
2. 智能体编排层：以状态图方式组织评审流程，支持并行评审、汇聚仲裁、重大缺陷人工介入。
3. 专业智能体层：采用“7+1”架构，包括电气一次、电气二次、土建、线路、规范合规、造价分析、安全评估、主审Agent。
4. 模型与知识引擎层：领域大模型、多模态模型、RAG知识库、知识图谱、历史评审意见库。
5. 工具与数据层：CAD/PDF解析、OCR、向量数据库、造价数据库、历史项目库、权限审计。

## 3. 7+1智能体设计

| 智能体 | 核心职责 | 关键输入 | 输出 |
|---|---|---|---|
| 电气一次Agent | 主接线可靠性、设备选型、短路电流、间隔配置一致性 | 一次接线图、设备表、短路计算书 | 设备参数校验、图纸定位、整改建议 |
| 电气二次Agent | 继电保护、自动化、二次回路、通信接口 | 保护配置表、二次图、通信方案 | 配置缺项、回路冲突、保护逻辑风险 |
| 土建Agent | 结构荷载、基础布置、抗震设防、坐标一致性 | 土建图、结构说明、地勘资料 | 坐标冲突、荷载风险、复核意见 |
| 线路Agent | 路径、杆塔、导线力学、交叉跨越 | 路径图、杆塔明细、线路说明 | 跨越风险、路径冲突、工程量异常 |
| 规范合规Agent | 强制性条文、行业规范、企业标准检索 | 设计成果文本、标准库 | 条文依据、适用条件、置信度 |
| 造价分析Agent | 工程量清单抽取、历史项目对比、异常检测 | Excel清单、设备数量、历史造价库 | 偏离比例、异常清单、参考项目 |
| 安全评估Agent | 安全距离、接地、消防通道、运维风险 | 总平面图、接地图、消防说明 | 风险等级、影响范围、整改优先级 |
| 主审Agent | 汇总、冲突仲裁、评分定级、报告生成 | 各Agent意见、项目元数据 | 结构化评审报告、闭环台账、审计记录 |

## 4. 典型评审流程

1. 项目创建：上传设计成果包，登记工程类型、电压等级、评审关注点。
2. 文件解析：解析DWG/PDF图纸、Word设计说明、Excel工程量清单。
3. 并行评审：各专业Agent并行抽取要素、检索规范、识别风险。
4. 交叉验证：主审Agent检查参数冲突、图纸与清单不一致、专业间边界问题。
5. 报告生成：输出带风险等级、证据链和整改建议的结构化报告。
6. 人工复核：专业负责人确认、修改或关闭意见，形成审计留痕。
7. 知识沉淀：将确认后的问题、整改意见和专家判断回流案例库。

## 5. 关键技术实现

### 5.1 多智能体编排

建议使用状态图方式编排：并行节点负责专业评审，汇聚节点负责主审仲裁，条件边负责重大问题人工介入。这样可以避免简单“多轮对话”导致流程不可控。

### 5.2 图纸理解

短期建议从PDF图纸、图签、设备表和清单抽取入手；中长期再做DWG图元拓扑解析。图纸理解应保留置信度，对低置信度字段进行人工复核。

### 5.3 规范RAG

规范知识库应以“条文编号、原文、适用范围、电压等级、标准版本、强制性标识”为核心字段。评审意见必须引用规范来源，禁止无依据结论进入正式报告。

### 5.4 造价异常检测

建立历史项目造价指标库，以项目类型、电压等级、地区系数、建设年代等作为特征，对工程量和单价进行对标。偏离阈值应结合企业经验动态配置。

### 5.5 知识图谱

推荐构建设备、规范、案例、专家四维知识图谱，把“设备参数—适用规范—历史问题—专家结论”连接起来，形成可复用的企业评审知识资产。

## 7. 关键治理原则

- AI意见必须有证据链：规范条文、图纸定位、抽取字段、模型置信度。
- 重大问题必须人工确认：AI不直接替代注册工程师或专业负责人。
- 知识库必须版本化：每次评审绑定规范库版本和模型版本。
- 流程必须可审计：保留模型调用、提示词模板、人工修改和审批记录。
- 从低风险场景试点：先做说明书、设备表、清单，再逐步进入复杂图纸。
